# Fase 1: Fundamentos do Express, Roteamento e CRUD em Memória

- [x] **1. O primeiro servidor**: Em uma pasta vazia, inicialize um projeto Node, instale o framework principal e crie um script `index.js` que apenas coloque uma aplicação para escutar na porta 3000, imprimindo uma mensagem de inicialização no terminal. `yarn init`
- [x] **2. Respostas estruturadas**: Crie uma nova API Express independente com uma única rota GET `/ping`. Esta rota não deve retornar texto puro, mas sim um objeto JSON com uma mensagem de "pong" e o status da operação. `res.json`
- [x] **3. Filtros dinâmicos**: Em um script isolado, construa uma rota GET `/calculator` que receba dois números opcionais via parâmetros na própria URL da requisição (ex: `?a=5&b=10`). Extraia esses valores, faça a soma e devolva o resultado no formato JSON. `req.query`
- [x] **4. Identificadores na rota**: Desenvolva uma aplicação Express do zero contendo uma rota GET `/profile/:username`. O servidor deve capturar o nome de usuário passado como trecho dinâmico e obrigatório do caminho da URL e devolver uma saudação personalizada. `req.params`
- [ ] **5. Automação do ambiente**: Configure um novo projeto Node.js, instale o Express e a ferramenta de reinicialização automática como dependência de desenvolvimento. Crie um atalho no `package.json` chamado `dev` e comprove o funcionamento criando uma rota `/status` que atualize a resposta sem precisar derrubar o servidor manualmente. `nodemon`
- [ ] **6. Recebendo pacotes de dados**: Levante uma nova API e configure o middleware nativo para interpretar requisições estruturadas. Crie uma rota POST `/echo` que receba um objeto JSON no corpo da requisição e simplesmente o devolva exatamente igual na resposta. `express.json`
- [ ] **7. Estruturando testes de API**: Crie um servidor Express simples com uma rota POST `/auth` que espera receber e-mail e senha. Abra o seu Client REST (como o Insomnia), configure uma requisição POST preenchendo o corpo JSON e faça o envio, utilizando um `console.log` na API para provar que a ferramenta enviou os dados corretamente. `client rest`
- [ ] **8. Persistência temporária**: Crie um arquivo independente contendo um servidor Express e uma constante com um Array vazio global. Implemente uma rota POST `/tasks` que receba um título de tarefa, insira no Array gerando um identificador sequencial, e retorne o registro recém-criado informando o código de status HTTP de sucesso adequado para criações. `array.push`
- [ ] **9. Retorno e Códigos de Status**: Inicie uma nova API contendo um Array fixo com 3 objetos fictícios de usuários (com IDs 1, 2 e 3). Construa uma rota GET `/users/:id` que busque o usuário. Se o ID existir no Array, retorne-o com status 200. Caso contrário, devolva um código de erro apropriado para recurso não encontrado, junto de um JSON de aviso. `status 404`
- [ ] **10. Projeto Portfólio: Gestão de Sustentabilidade**: Crie totalmente do zero um projeto de API RESTful focado no mapeamento de um projeto de extensão universitária sobre inovação e sustentabilidade. O sistema deve usar um Array em memória para gerenciar o recurso "Membros da Equipe". Cada membro possuirá um ID gerado automaticamente, nome e o curso associado (Jornalismo, Psicologia ou Computação). Implemente todas as rotas CRUD (listar todos, buscar por ID, cadastrar pelo corpo da requisição, atualizar dados buscando pelo índice e excluir permanentemente). O projeto deve inicializar seu próprio servidor, usar automação de reinicialização, ter rotas semânticas e aplicar os status HTTP corretos em cada interação. `crud restful`
