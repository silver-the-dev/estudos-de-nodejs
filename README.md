# Fase 1: Fundamentos do Express, Roteamento e CRUD em Memória

- [x] **1. O primeiro servidor**: Em uma pasta vazia, inicialize um projeto Node, instale o framework principal e crie um script `index.js` que apenas coloque uma aplicação para escutar na porta 3000, imprimindo uma mensagem de inicialização no terminal. `yarn init`
- [x] **2. Respostas estruturadas**: Crie uma nova API Express independente com uma única rota GET `/ping`. Esta rota não deve retornar texto puro, mas sim um objeto JSON com uma mensagem de "pong" e o status da operação. `res.json`
- [x] **3. Filtros dinâmicos**: Em um script isolado, construa uma rota GET `/calculator` que receba dois números opcionais via parâmetros na própria URL da requisição (ex: `?a=5&b=10`). Extraia esses valores, faça a soma e devolva o resultado no formato JSON. `req.query`
- [x] **4. Identificadores na rota**: Desenvolva uma aplicação Express do zero contendo uma rota GET `/profile/:username`. O servidor deve capturar o nome de usuário passado como trecho dinâmico e obrigatório do caminho da URL e devolver uma saudação personalizada. `req.params`
- [x] **5. Automação do ambiente**: Configure um novo projeto Node.js, instale o Express e a ferramenta de reinicialização automática como dependência de desenvolvimento. Crie um atalho no `package.json` chamado `dev` e comprove o funcionamento criando uma rota `/status` que atualize a resposta sem precisar derrubar o servidor manualmente. `nodemon`
- [x] **6. Recebendo pacotes de dados**: Levante uma nova API e configure o middleware nativo para interpretar requisições estruturadas. Crie uma rota POST `/echo` que receba um objeto JSON no corpo da requisição e simplesmente o devolva exatamente igual na resposta. `express.json`
- [x] **7. Estruturando testes de API**: Crie um servidor Express simples com uma rota POST `/auth` que espera receber e-mail e senha. Abra o seu Client REST (como o Insomnia), configure uma requisição POST preenchendo o corpo JSON e faça o envio, utilizando um `console.log` na API para provar que a ferramenta enviou os dados corretamente. `client rest`
- [x] **8. Persistência temporária**: Crie um arquivo independente contendo um servidor Express e uma constante com um Array vazio global. Implemente uma rota POST `/tasks` que receba um título de tarefa, insira no Array gerando um identificador sequencial, e retorne o registro recém-criado informando o código de status HTTP de sucesso adequado para criações. `array.push`
- [x] **9. Retorno e Códigos de Status**: Inicie uma nova API contendo um Array fixo com 3 objetos fictícios de usuários (com IDs 1, 2 e 3). Construa uma rota GET `/users/:id` que busque o usuário. Se o ID existir no Array, retorne-o com status 200. Caso contrário, devolva um código de erro apropriado para recurso não encontrado, junto de um JSON de aviso. `status 404`
- [x] **10. Projeto Portfólio: Gestão de Sustentabilidade**: Crie totalmente do zero um projeto de API RESTful focado no mapeamento de um projeto de extensão universitária sobre inovação e sustentabilidade. O sistema deve usar um Array em memória para gerenciar o recurso "Membros da Equipe". Cada membro possuirá um ID gerado automaticamente, nome e o curso associado (Jornalismo, Psicologia ou Computação). Implemente todas as rotas CRUD (listar todos, buscar por ID, cadastrar pelo corpo da requisição, atualizar dados buscando pelo índice e excluir permanentemente). O projeto deve inicializar seu próprio servidor, usar automação de reinicialização, ter rotas semânticas e aplicar os status HTTP corretos em cada interação. `crud restful`

# Fase 2: Arquitetura MVC, Sucrase e Padronização de Código

- [x] **11. Depuração Profissional**: Crie uma API simples com o framework Express contendo uma rota GET. Em vez de usar `console.log` para encontrar erros, configure o depurador nativo do **Visual Studio Code** (criando a configuração para Node.js no arquivo `launch.json`). Insira um _Breakpoint_ (ponto de parada) clicando ao lado da linha da rota, execute o servidor pelo modo "Debug" do VS Code e faça uma requisição pelo Insomnia para ver o código pausar exatamente na linha marcada. `vs code debug`
- [x] **12. Servidor Orientado a Objetos**: Inicialize um projeto Node.js do zero. Crie uma pasta chamada `src`. Dentro dela, crie um arquivo `app.js` estruturado como uma **Classe JavaScript** (ex: `class App`). O método construtor dessa classe deve ser o único responsável por instanciar o framework Express. Em seguida, crie um arquivo `server.js` isolado que apenas importe essa classe e coloque a aplicação para escutar na porta 3000. `class App`
- [x] **13. Roteamento Isolado**: Em um novo projeto do zero contendo a estrutura de classe do exercício anterior, remova qualquer definição de rota do arquivo principal. Crie um arquivo `src/routes.js` e utilize a ferramenta **Express.Router()**. Defina uma rota GET `/status` neste arquivo, exporte-a e importe-a dentro da sua classe `App`, conectando-a ao servidor através do comando `.use()`. `Express.Router`
- [x] **14. Introdução a Controllers**: Comece uma nova API independente. Crie a pasta `src/app/controllers` e adicione um arquivo chamado `UsersController.js`. Desenvolva uma classe contendo um método chamado `index` que retorne um objeto JSON. No seu arquivo `routes.js`, remova a lógica da resposta e faça com que a rota GET `/users` apenas aponte para o método `UsersController.index`. `Controller`
- [ ] **15. Mapeamento de Métodos CRUD**: Inicialize um novo projeto. Crie um `TasksController.js`. Escreva a estrutura vazia dos 5 métodos padrões de um Controller RESTful ensinados no curso: `index` (listar), `show` (exibir um), `create` (criar), `update` (atualizar) e `destroy` (deletar). Faça com que cada método retorne apenas um JSON com uma mensagem simulando a ação (ex: "Criando tarefa"). Mapeie todos eles para suas respectivas rotas e verbos HTTP no arquivo `routes.js`. `Métodos padrão`
- [ ] **16. Sintaxe Moderna (ES Modules)**: Crie um projeto do zero e instale o Express. Substitua o padrão antigo do Node (`require` e `module.exports`) pela sintaxe moderna (`import` e `export default`). Como o Node.js não interpreta isso nativamente de forma direta, instale a biblioteca **Sucrase** como dependência de desenvolvimento (`yarn add sucrase -D`). Rode o servidor no terminal usando o comando `sucrase-node src/server.js`. `import / export`
- [ ] **17. Automação com Sucrase**: Em uma nova pasta vazia, configure um projeto instalando o framework Express, o **Sucrase** e o **Nodemon**. Na raiz do projeto, crie um arquivo chamado `nodemon.json`. Nele, crie uma chave `execMap` informando ao Nodemon que todos os arquivos `.js` devem ser interceptados e executados pelo comando `sucrase-node`. Crie um script no manifesto (`package.json`) para inicializar o ambiente de forma prática. `nodemon.json`
- [ ] **18. Padronização de Ambiente**: Crie um novo projeto vazio. Instale a extensão **EditorConfig** no seu editor de código. Na raiz do projeto, gere o arquivo `.editorconfig`. Configure as regras universais do projeto para forçar indentação com 2 espaços (`indent_size = 2`), uso do charset `utf-8`, inserção de uma linha vazia no final do arquivo e remoção automática de espaços em branco invisíveis no final das linhas (`trim_trailing_whitespace = true`). `.editorconfig`
- [ ] **19. Qualidade de Código e Linting**: Inicialize um projeto Node.js. Instale as bibliotecas **ESLint** e **Prettier** como dependências de desenvolvimento. Rode o comando de inicialização do ESLint (`yarn eslint --init`) e escolha adotar o guia de estilo padrão da empresa **Airbnb**. Escreva um código mal formatado (ex: sem ponto e vírgula ou com aspas duplas) e veja o editor alertar os erros. Configure o editor para corrigir esses problemas automaticamente ao salvar o arquivo (usando o _autofix_). `eslint --init`
- [ ] **20. Projeto Portfólio: Boilerplate MVC Profissional**: Crie do zero absoluto a base estrutural perfeita para qualquer futura API. O projeto deve usar a estrutura de pastas `src/app/controllers`, ter a inicialização orientada a objetos (Classe `App`) e usar o roteamento isolado (`routes.js`). Crie um `ProjectsController` com os 5 métodos CRUD retornando mensagens simuladas. Aplique a sintaxe moderna (`import/export`) compilada com **Sucrase** e automatizada com **Nodemon**. Garanta que o projeto contenha os arquivos `.editorconfig`, `.eslintrc.js` e `.prettierrc` rodando em harmonia, sem que o terminal ou o editor apontem nenhum erro de sintaxe. `boilerplate mvc`
